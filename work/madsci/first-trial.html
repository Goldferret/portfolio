First Successful Robot Trial

June 17, 2026

Today the tri-agent system moved a physical robot for the first time in an autonomous experiment. Four colored blocks — two red, two blue — were placed in a detection zone in front of a DOFBOT Pro arm. The goal: sort them by color into two goal zones without any human intervention beyond pressing start.

It worked. Partially.

What happened

The Detector Agent ran a computer vision pipeline using custom HSV color thresholds, identified the blocks in the camera frame, and returned pixel coordinates and depth estimates for each detection. The Executor Agent received those detections, reasoned about which block belonged in which goal zone based on the assigned_colors field in the procedure, and issued pick-and-place commands to the robot arm.

The robot moved. That was the milestone.

The trial surfaced a few expected rough edges: the detection pipeline found three of the four blocks rather than all four, likely a lighting or threshold issue. The Executor initially attempted a pick_and_place workflow action that doesn't exist, caught its own mistake, called get_node_info to discover the correct action names (pick_at_pixel_coords, place_at_pixel_coords), and recovered. Two of the four blocks ended up in the correct goal zones.

What we learned

The self-correction behavior — try, fail, introspect, retry — is not a bug. It's the core behavior we're studying. In the tri-agent condition, the Executor has a narrow toolset and a pre-built detection procedure, so its failure modes are constrained and interpretable. The wasted actions from self-correction count toward our redundant actions metric and are part of what we compare against the single-agent and scripted baselines.

Detection reliability is the more pressing issue. A 75% detection rate (3 of 4 blocks) means some trials will fail before the robot ever picks anything up. We need to either tighten the block placement protocol or adjust the HSV thresholds for the lab lighting conditions.

What's next

  • Fix detection to reliably find all 4 blocks under lab lighting
  • Run 30 trials for the tri-agent condition to collect the experiment dataset
  • Build the single-agent baseline (all 48 tools, generic prompt) and run 30 trials
  • Build the scripted baseline and run 30 trials
  • Analyze: task success rate, completion rate, execution time, total actions, redundant actions, replanning events

The paper target is IROS 2026 Workshop. We have time, but the robot needs to cooperate.
