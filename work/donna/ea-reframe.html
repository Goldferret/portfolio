The EA reframe

Donna — v2.0 — May 2026

The v1 system worked. Morning briefs ran, todos generated, emails scanned. But somewhere around month three, the pattern became clear: I was still the one managing the system. I had to remember to run @todos. I had to notice when something needed capturing. The AI was organized and responsive — it just wasn't proactive.

The reframe came from a conversation with Perplexity about why this was happening. The answer came from cognitive psychology: what I was experiencing has a name — prospective memory failure. The breakdown of intention-to-act when there's no external cue at the moment of execution. A personal assistant that waits to be asked is still fundamentally the same as a todo app. I needed something closer to how a real executive assistant operates: someone who captures without being asked, who surfaces the right thing at the right boundary, who maintains context across sessions without having to be reminded of it.

The architecture needed to change in three ways.

The first was the schema. v1 used a loose daily note system with no real data model underneath. v2 introduced a proper GTD schema: individual notes with frontmatter for ticklers, waiting-for items, agenda items, and someday-maybe. Each type is now queryable independently — obsidian_query(where={"type": "tickler", "status": "pending"}) returns exactly what it sounds like. The system can now answer "what's overdue?" or "what am I waiting on from a collaborator?" from structured data rather than prose scanning.

The second was the focus filter. The focus spec (focus-spec.md) and calendar (calendar.md) gave the system a machine-readable decision layer: Summer Sprint, Fall Semester, Winter Break, each with explicit scope rules. When something new surfaces, the system checks whether it belongs in the current phase before surfacing it. That one addition changed how context gets routed and how the todo list gets generated — scope-checking became automatic rather than something I had to do mentally.

The third was the morning fleet — something v1 didn't have at all. A launchd-scheduled headless agent runs every morning before I open a session, writes a pre-computed brief to the vault, and the main session reads it on initialization. The fleet is a coordinator of specialist skills with defined output contracts: email-scan, calendar-phase, session-journal, project-state. Each skill does one thing and returns a structured summary. I no longer open a chat to a blank slate.

The behavioral protocols — proactive task capture, waiting-for logging, topic-transition surfacing — were specified as the design target for this layer. The morning fleet implements them for session boundaries: it checks for overdue items, surfaces waiting-for threads, and pre-loads context so the session starts informed. Real-time detection during active sessions — capturing a task the moment it surfaces in conversation, without being asked — is still the open problem.

The system went live May 19, 2026. The entry in that day's log: "Designed and implemented the full Donna v2 EA system architecture."
